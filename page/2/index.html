ShadowSocks服务器搭建iptables配置

发表于 2015-01-26 | 更新于 2015-04-14

设定默认规则：

iptables -P INPUT  DROP
iptables -P OUPUT ACCEPT
iptables -P FORWAD ACCEPT

允许SSH：

iptables -A INPUT -p tcp --dport 22 -j ACCEPT

允许Shadowsocks端口：

iptables -A INPUT -p tcp --dport 8388 -j ACCEPT

允许合法链接并丢弃非法连接：

iptables -I INPUT -m state --state NEW,ESTABLISHED,RELATED -j ACCEPT
iptables -I INPUT -m state --state INVALID -j DROP

保存或恢复iptables规则

iptables-save > somefile
iptables-restore < somefiles

魅族Flyme开启USB调试模式之后，无法在Ubuntu等系统下识别的解决方法

发表于 2015-01-06

Ubuntu环境

  1. 建立或修改文件 ~/.android/adb_usb.ini，在文件开头或末尾添加一行，内容是0x2a45。
  2. 建立或修改文件 /etc/udev/rules.d/51-android.rules
    2.1 修改文件权限：
    sudo chmod a+rx /etc/udev/rules.d/51-android.rules
    2.2 在文件中添加一行内容：
    SUBSYSTEM=="usb", ATTR{idVendor}=="2a45", MODE="0666", GROUP="plugdev"

  3. 确保你在用户组”plugdev”
    sudo adduser plugdev

  4. 按下面流程重启udev服务或直接重启PC
    sudo service udev restart

  5. 检查ADB设备是否存在
    adb kill-server
    adb devices

Windows XP中文环境

  1. 建立或修改C:\Documents and Settings\<你的用户名>\.android\adb_usb.ini文件，在该文件中添加一行文本，内容是0x2a45。

  2. 检查ADB设备是否找到（命令行程序中）
    adb kill-server
    adb devices

Windows 7中文环境

  1. 建立或修改C:\用户\<你的用户名>\.android\adb_usb.ini文件，在该文件中添加一行文本，内容是0x2a45。

  2. 检查ADB设备是否找到（命令行程序中）
    adb kill-server
    adb devices

参考文档： http://developer.android.com/tools/device.html
来源：MX4_ADB_参考说明书.txt（内置光盘模式）
