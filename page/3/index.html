ShadowSocks服务器搭建iptables配置

发表于 2015-01-26 | 更新于 2015-04-14

设定默认规则：

iptables -P INPUT  DROP
iptables -P OUPUT ACCEPT
iptables -P FORWAD ACCEPT

允许SSH：

iptables -A INPUT -p tcp --dport 22 -j ACCEPT

允许Shadowsocks端口：

iptables -A INPUT -p tcp --dport 8388 -j ACCEPT

允许合法链接并丢弃非法连接：

iptables -I INPUT -m state --state NEW,ESTABLISHED,RELATED -j ACCEPT
iptables -I INPUT -m state --state INVALID -j DROP

保存或恢复iptables规则

iptables-save > somefile
iptables-restore < somefiles

魅族Flyme开启USB调试模式之后，无法在Ubuntu等系统下识别的解决方法

发表于 2015-01-06

Ubuntu环境

  1. 建立或修改文件 ~/.android/adb_usb.ini，在文件开头或末尾添加一行，内容是0x2a45。
  2. 建立或修改文件 /etc/udev/rules.d/51-android.rules
    2.1 修改文件权限：
    sudo chmod a+rx /etc/udev/rules.d/51-android.rules
    2.2 在文件中添加一行内容：
    SUBSYSTEM=="usb", ATTR{idVendor}=="2a45", MODE="0666", GROUP="plugdev"

  3. 确保你在用户组”plugdev”
    sudo adduser plugdev

  4. 按下面流程重启udev服务或直接重启PC
    sudo service udev restart

  5. 检查ADB设备是否存在
    adb kill-server
    adb devices

Windows XP中文环境

  1. 建立或修改C:\Documents and Settings\<你的用户名>\.android\adb_usb.ini文件，在该文件中添加一行文本，内容是0x2a45。

  2. 检查ADB设备是否找到（命令行程序中）
    adb kill-server
    adb devices

Windows 7中文环境

  1. 建立或修改C:\用户\<你的用户名>\.android\adb_usb.ini文件，在该文件中添加一行文本，内容是0x2a45。

  2. 检查ADB设备是否找到（命令行程序中）
    adb kill-server
    adb devices

参考文档： http://developer.android.com/tools/device.html
来源：MX4_ADB_参考说明书.txt（内置光盘模式）

BuildAPP 超过65535个方法的解决方法

发表于 2014-12-18

自Android5.0发布之后，官方给出了Multidex工具包，该工具包在5.0及以上的系统版本是不需要单独引入的，但是如果使用5.0之前的系统，需要单独引入。
引入方法：

  • 修改配置build.gradle

    android {
    compileSdkVersion 21
    buildToolsVersion "21.1.1"

    defaultConfig {
    ...
        minSdkVersion 14
        targetSdkVersion 21
    ...

       // 开启 multidex 
       multiDexEnabled true
    }
    ...
}

dependencies {
   compile 'com.android.support:multidex:1.0.0'
}

  • 修改AndroidManifest.xml，自定义application为android.support.multidex.MultiDexApplication，如果是已经有自定义application则直接看下一步。

    <?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.example.android.multidex.myapplication">
    <application
        ...
        android:name="android.support.multidex.MultiDexApplication">
        ...
    </application>
</manifest>

  • 如果是自定义过application 则继承MultiDexApplication。
    网上有资料还要加一步骤 MultiDex.install(getTargetContext());其实这一步骤是没必要的，MultiDexApplication已经做了相关操作。

但是根据实际使用Multidex仅支持4.0及以上系统，2.3会无法安装。

如果用户群体中4.0以下的用户比较多的话，目前还是建议精简自己的代码和做混淆压缩优化。

自com.android.tools.build:gradle:0.14.0开始，runProguard 更名为minifyEnabled，并且支持自动去除没有使用的资源，达到减小APK体积，去除没用资源默认是关闭的，需要通过设置build type中的shrinkResources true打开，此功能需要依赖代码”瘦身“，即保证minifyEnabled true。具体代码

buildTypes {
    release {
        minifyEnabled           true
        shrinkResources         true
    }
}

Multidex说明官方原文：https://developer.android.com/tools/building/multidex.html

Gradle构建Android项目NDK集成路径问题

发表于 2014-11-20

用Gradle 构建Android项目时，jniLibs应该放在

src/main/jniLibs/<abi>/

例如

src/main/jniLibs/armeabi-v7a/lib_hello.so
src/main/jniLibs/x86/lib_hello.so

或者自定义路径（修改build.gradle）

android {
    sourceSets.main {
        jniLibs.srcDirs = ['native-libs']
    }
}
